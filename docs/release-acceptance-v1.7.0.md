## 非技术用户主路径

由未参与开发的人从干净环境开始，只提供发布包和一份脱敏案件：

- [ ] 仅用自然语言「整理案件资料」启动 build，无需人工拼接内部命令。
- [ ] 所有原始材料均进入已产出、带理由跳过或未处置三态之一。
- [ ] 转换失败、OCR 可疑和外部工具缺失均清楚暴露，没有静默完成。
- [ ] 随机抽查金额、日期、姓名、条款，Wiki 事实能逐字回到来源文本和原始材料。
- [ ] 提问一项材料明确记载的事实，回答带有效锚点。
- [ ] 提问一项材料不存在的事实，回答严格为「未在本案材料中找到」。
- [ ] 构造或选取一项来源冲突，系统并列证据并把最终判断交给人。
- [ ] 使用一份 `quality: suspect` 来源，所有引用保留「（未核验）」标记。
- [ ] 关闭 RAG 后重新提问，系统明确降级且 Wiki/grep/outline 路径仍可用。
- [ ] 未给予云端同意时，OCR、标题重建和远程 embedding 均不外发。
