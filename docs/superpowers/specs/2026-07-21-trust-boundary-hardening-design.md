# 可信边界加固设计（2026-07-21）

## 背景

v1.6.0 已具备逐字锚点、覆盖账本、显式 OCR 同意和 parent-context，但审查发现若干“局部检查通过、整体承诺未闭合”的路径。本批次只加固既有边界，不新增业务能力。

## 决策

1. **答案逐行接地**：分析 callout 之外的每个实质内容行必须含来源锚点，或明确表述未找到。一个合法锚点不能替其他裸事实免责；普通 blockquote 不算分析。
2. **统一闭世界**：wiki ingest 与 answer 使用同一套解析后路径包含检查，来源必须真实位于本案 `_md/`。
3. **可验证派生关系**：makeitdown 写入 `provenance_version: 1`、原件与正文 SHA-256；lint 对版本化产物 fail-closed 复核。旧产物保持可读，但以 warning 阻断“ingest 完成”，重转后进入强校验。
4. **完整性警告阻断完成**：覆盖率仍以 warning 展示，但 `lint check` 返回非零。本决策取代 2026-07-12 coverage-ledger 设计中的 soft-exit 约定。
5. **统一数据外发同意**：云 OCR、MinerU、标题 LLM 与远端 embedding 都进入显式 consent 边界。
6. **自动化失败可观察**：安装失败返回非零；输出与 JSON sidecar 原子替换。
7. **稳定检索契约**：每个 hit 固定包含 `parent_text`；配置拒绝 NaN/Inf/越界阈值；长期进程在 sidecar 更新后重载父块。

## 非目标

- 不在本批次实现 GUI、用户权限系统、加密存储或完整电子证据签名体系。
- 历史无版本/无哈希 `_md` 可继续读取，但不能通过完成闸门；迁移通过重转完成。
- sidecar 热加载降低单进程陈旧风险，但多写者的完整事务/锁仍留作后续存储架构工作。

## 验收

- 新增绕过、路径穿越、哈希变化、云端同意、退出码、NaN/Inf、parent contract 和热加载测试。
- 三个子项目全量测试与仓库 Ruff `E9,F` 门禁通过。
