# 校验（ingest 第 8、9 步的细节）

> 校验工具随本 skill 发布：本 skill 目录的 `lint/lint.py`（仅标准库，需 Python 3.11+）。`<SKILL_DIR>` 指本 skill 实际所在目录（agent 加载本 skill 时已知）。案件目录与 skill 目录通常不同，**务必用 `<SKILL_DIR>/lint/lint.py` 的绝对路径调用**。
>
> 8 管"引文真"，9 管"断言不脑补"。两关都过才算这次 ingest 完成。

## 第 8 步 · 确定性校验

`python <SKILL_DIR>/lint/lint.py check <案件根目录>`（案件根目录下有 `wiki/` 与 `_md/`）。退出码非 0 即有**违规**，必须修到 0。

**违规（hard）：**
1. **锚点存在 + 闭世界**：每个逐字片段出现在所指来源里，且解析后的路径真实位于本案 `_md/`；`..` 与符号链接逃逸都会被拒绝。
2. **来源完整性**：`provenance_version: 1` 的 makeitdown 产物必须同时具备 `source`、`source_sha256`、`content_sha256`，并分别与原件和转换正文一致；任一缺失或被改即报错。无版本旧产物可读，但 warning 未清零前不得宣称完成。
3. **死链**：每个 `[[wikilink]]` 解析到真实页面或其别名。
4. **时间线顺序**：时间线页内可解析日期非递减。
5. **勾稽闭合**：`> [!check] a + b == c` 断言的算术成立（安全求值，只许 +−× 与数字，绝不 eval；等式后不得追加说明或事实，说明另起分析行）。

**完整性警告（显示为 warning，但影响退出码）：**
6. **覆盖率（三态账本）**：`_md/` 下每个源文件必居其一——**已引用**、**登记跳过**、**未处置**。未处置或 skip 缺非空原因时 `check` 返回非零，防止 agent/脚本误报完成；条目路径不存在则忽略，而真实未处置文件仍会暴露。

Wiki 页面首标题只有在与文件名一致（页面身份）或属于固定结构标题时才豁免；事实仍须写在正文并挂锚点。有锚点的事实行在最后一个锚点后只能保留标点、表格边界或「（未核验）」标记，不能继续夹带另一项事实。

校验**只消除格式噪声**（空白/换行/全半角标点/千分位逗号/markdown/表格/HTML 标签），**数字与文字精确**——"换行差异"不误报，"数字写错/张冠李戴"必被抓。这是把 EXTRACTED 从自觉变成可机检的那一步。

### 源级对账（覆盖率的补盲，收尾单独跑）

覆盖率账本只统计 `_md/` 下的文件，**看不见转换失败/跳过、从未进入 `_md/` 的源文件**。`python <SKILL_DIR>/tools/reconcile.py <案件根目录>` 补这个盲：读 `_md/report.json`，对源级非产出做对称账本并核对原始文件总数。两道账本的**未处置 > 0 均返回非零**。源级 skip 与 `_md` 级 skip 共用 `wiki/log.md`，靠路径前缀区分。

源级 skip 同样必须带非空「原因」；只有路径、没有理由的登记会报 `[跳过无原因]`，继续阻断完成。

## 第 9 步 · 蕴含校验（claim↔引文，换实例判官）

锚点 lint 只证明"引文存在"；蕴含校验进一步证明"**断言真能由引文推出**"，抓"引文真、但断言被脑补/拔高/歪曲"（认购→已实缴、拟→已、部分→全体、指控→认定）。**每次 ingest 收尾自动走：**

1. `python <SKILL_DIR>/lint/lint.py extract <案件根目录> > worklist.json` —— 拆出每条 `(断言, 引文, 来源, 源文上下文)`（每锚点配它紧前的子断言）。
2. **派一个全新子代理当判官**（绝不让本 ingest agent 判自己）：只喂这些对的"断言 + 引文 + 上下文"、不喂全文、不许外部知识，按下方 prompt 三分判。
3. **修**（由 ingest agent 做，不是判官）：
   - 不支持 → 把断言改成忠实于引文的说法，或降级为 `> [!note] 分析`；
   - 信息不足 → 换一条能完整支持的引文、或按"一断言一引文"拆开、或标「未核验」。
   - **只许把断言改忠实，绝不为凑通过而编造引文里没有的内容**（那是把幻觉洗白）。
4. 改完**重跑第 8 步 + 重判变动项**，最多三轮。
5. 三轮仍判不过的，**连同判官理由显著列给用户**复核，不静默通过。

判官是**筛子不是自动放行**；修复由 ingest agent 有界进行、未解决必上报。

### 判官 prompt（照用）

> 你是严格的法律文书引文蕴含判官。仅凭给定的**引文（及其源文上下文窗口）本身**判断能否支持断言，不许用外部知识或其他对的信息替断言圆场。每条三选一：
> - **支持**：引文直接、完整支持断言，断言没有超出引文。
> - **不支持**：引文与断言矛盾，或断言把引文歪曲/拔高（认购/价款≠已实缴/已转账；拟≠已；部分≠全体；方向反了；指控≠认定）。
> - **信息不足**：引文为真，但不足以支持断言全部（断言含引文未提到的人数/合计/"全体一致同意"等，或把"现有/某时点"说成"设立时"等不同时点）。
> 输出：表格（序号|判定|一句话理由）+ 单列所有"不支持/信息不足"的序号作为待复核清单。只判定，不改写、不建议。

### 局限

- 是**网不是保证**；必须换实例；上下文窗口太小会饿死、太大会放水。
- 抓不了**遗漏**（断言忠实但不完整）——归覆盖率/人工。
- 法律**定性是否正确**仍归人。

**写作铁规则**：**一条断言 ← 一条能完整支持它的引文**。别把多个事实（日期+轮次+金额+注册资本…）塞在一条锚点下；一句里有几个独立事实就挂几条各自够用的锚点。
