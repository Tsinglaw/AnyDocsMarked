# lawiki 覆盖率三态账本设计

> **状态：已实现，部分约定已取代。** 三态账本继续有效；“warning 不影响退出码”的旧约定已被 2026-07-21 可信边界设计取代：未处置或无理由跳过会阻断“ingest 完成”。

> 日期：2026-07-12
> 原始状态：已与用户逐层确认，待实现
> 背景事件：《文档处理偏差说明与问题汇报》（炎凰数据案）——agent 在 73 个 `_md/` 源文件中仅 ingest 5 个，将 68 个静默降格为 backlog，事后包装成"设计如此"。lint 如实报出 68 条覆盖率警告，但警告不区分"漏了"与"故意跳过"、无法收敛，实践中沦为背景噪声。

## 一、问题定性

机制层设计经受住了考验：锚点/三类标注/蕴含校验全部守住，agent 未编造任何内容；覆盖率 soft 警告如实发出。悖于设计初衷的是意图层：

1. **"交人判断"变成了"agent 自判"**——verification.md 把"漏 ingest 还是有意跳过"的判断权交给 agent/人，初衷是人做最终裁决；实际是 agent 静默行使了裁决权，从未把范围决策端到用户面前。
2. **事后合理化与 rag.md 相悖**——"wiki=骨干 / RAG=细节"的说法把 `.rag/` 降格为兜底仓，而设计明确 `.rag/` 与 `wiki/` 平级、互为交叉验证；大量材料只进 RAG 不进 wiki，恰恰架空交叉验证。

根因：**设计信任了判断，却没有要求判断留痕。** 覆盖率警告只有两态（引用/未引用），没有"已登记为有意跳过"的第三态，导致：(a) 即使 agent 做了正确取舍，警告也永不清零，必然被当噪声；(b) "0 硬违规"被 Goodhart 成"完成"的代理指标——少写就少错，写 5 页比 73 页更容易拿到退出码 0。

## 二、目标

1. 覆盖率警告**可收敛到 0**：一个"完成"的案件应零警告，警告重新变成有信息量的信号。
2. 跳过决策**显式留痕、可审计**：用户一眼看到 agent 替他做了哪些取舍及理由。
3. 堵死"0 违规 = 完成"通道：明确定义 ingest 完成 = 0 硬违规 **且** 覆盖率未处置 = 0。

## 三、不变量（不动的东西）

- 覆盖率保持 **soft**，不影响退出码；四类 hard 闸门（锚点/死链/时间线/勾稽）不动。
- lint 仅标准库（Python 3.11+）不变。
- wiki 固定结构不变，**零新增文件**——账本复用 `log.md`。
- 零误报原则：新增的所有判据均为确定性格式判据。

## 四、`log.md` skip 条目（新约定）

沿用 log.md 既有条目形态 `## [YYYY-MM-DD] <操作> | <对象>`，新增 `skip` 操作：

```markdown
## [YYYY-MM-DD] skip | _md/<相对路径>
- 原因：<一句非空理由>
```

- **路径**：相对案件根的 POSIX 路径（`_md/…`），与锚点、覆盖率检查同一坐标系。
- **原因**：条目正文中一行 `- 原因：<非空文本>`，必填（缺失触发 soft 警告，见下）。
- **撤销跳过无需删条目**：文件一旦被任何锚点引用即归"已引用"——**引用优先于登记**，log.md 的 append-only 性质不破。
- **不设"待补"登记态**：未处置警告本身就是 backlog 信号。若允许登记"待补"来消警告，等于重新打开静默降格通道。

## 五、lint 三态语义（`_check_coverage` 改造）

新增解析：扫 `wiki/log.md` 中匹配 `## [YYYY-MM-DD] skip | <路径>` 的标题，收集登记集合，并检查各条目正文是否含非空 `- 原因：`。

对 `_md/` 下每个 `.md`，三态必居其一：

| 态 | 判据 | 输出 |
|---|---|---|
| 已引用 | 被任何锚点引用（现有 `cited` 集合） | 无 |
| 登记跳过 | 未引用，但在 skip 登记集合中 | 无 |
| 未处置 | 两者皆非 | soft 警告 `[未处置] <路径>`（原 `[未引用]` 更名） |

附加输出与判据：

- **汇总行恒输出**（有无警告都打印）：`覆盖率：<N> 源文件 | 已引用 <a> | 登记跳过 <b> | 未处置 <c>`。这行是"当前账目一览"，由 lint 现算，永不漂移——不引入需要手工维护的第二事实源。
- **skip 条目缺原因或原因为空** → soft 警告 `[跳过无原因] <路径>`（格式即判据，零误报）。
- **skip 路径在 `_md/` 中不存在** → 静默忽略。不会造成漏网：路径写错时目标文件仍以未处置警告暴露；文件后来被删除时条目自然失效、不制造噪声。
- 退出码语义不变：覆盖率相关输出均不影响退出码。

## 六、SKILL.md 第三步措辞修补

1. "对 `_md/` 下每个 `.md`" 仍是默认目标，措辞不弱化。
2. 新增：**允许分批 / 脊柱优先策略，但必须向用户申报范围**——每轮 ingest 开始或结束时报"本轮 ingest n / 登记跳过 m / 待补 k"；待补清零前不得宣称 ingest 完成。
3. 新增完成定义：**ingest 完成 = lint 0 违规 且 覆盖率未处置 = 0**（每个源文件要么被引用、要么登记跳过）。
4. 跳过必须写 skip 条目并附原因；不许以"记入 backlog"等形式静默降格。

## 七、文档同步

- `references/verification.md`：第 5 类覆盖率改写为三态表述（判据、汇总行、`[未处置]`/`[跳过无原因]` 警告、引用优先于登记、无"待补"态及其理由）。
- `references/page-formats.md`：log.md 一节补 skip 条目格式与示例。
- `lint/lint.py` 模块 docstring 中"⑤ 覆盖率（警告）"的描述同步。

## 八、测试（lint pytest 新增用例）

1. 未引用且未登记 → `[未处置]` 警告。
2. 未引用但已登记（含原因）→ 无警告。
3. 已引用且已登记 → 无警告（引用优先，登记冗余无害）。
4. 登记条目缺 `- 原因：` 或原因为空 → `[跳过无原因]` 警告。
5. 登记路径在 `_md/` 不存在 → 静默忽略，无警告。
6. 汇总行数字正确（N = a + b + c）。
7. 退出码：仅覆盖率警告时仍为 0。

## 九、明确不做

- 覆盖率升 hard：账本机制下"未处置"判据零误报，理论上有资格升 hard，但先以 soft + 账本观察一两个案件后再由用户裁决。
- "待补 / 进行中"登记态（见第四节）。
- 独立账本文件（`wiki/覆盖率.md`、`.lawiki-skip` 等）：前者是需手工维护的第二事实源，后者把决策藏在人看不见的地方，均已否决。
