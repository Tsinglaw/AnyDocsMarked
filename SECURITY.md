# Security Policy

AnyDocsMarked 面向可能含敏感案卷的本地工作流。安全问题包括但不限于：未经同意的数据外发、路径穿越、跨案件数据混用、锚点/哈希校验绕过、密钥泄漏以及恶意文档导致的代码执行。

## 报告漏洞

请优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口，不要在公开 Issue 中附真实案件、token、原始文档或可识别个人的信息。

报告请包含：受影响版本、最小复现、预期/实际行为、影响范围，以及已经做过的脱敏处理。维护者确认前，请勿公开利用细节。

## 支持范围

安全修复优先落在最新发布版本；旧版本用户通常需要升级。当前项目仍处于早期 Beta，正式案卷工作流必须保留人工复核、最小权限和独立备份。

## 数据处理基线

- 默认优先本地后端；使用外部 OCR、LLM、embedding 或 reranker 前确认服务商与数据范围。
- token 只放环境变量或受控密钥存储，不写入仓库、命令历史、报告或案件 wiki。
- 不把真实案件材料提交到测试夹具、Issue、CI artifact 或公共模型评测集。
- `_md/`、`wiki/`、`.rag/` 和日志按案件隔离；交付或删除前核对所有派生副本。
